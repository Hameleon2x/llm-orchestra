# Справочник по опциям прогона

Полный разбор [`Agent\Dto\RunOptions`](../../src/Agent/Dto/RunOptions.php) — параметров одного вызова `Runner::run()`. Настройки самих моделей живут в каталоге, см. [02-catalog-and-fallback.md](02-catalog-and-fallback.md).

## Откуда берутся значения

Опции — аргумент вызова, а не конфигурация приложения: объект создаётся на каждый прогон и живёт столько же. Значения по умолчанию для всех прогонов задаются секцией `defaultRun` каталога:

```php
$registry = Registry::fromArray([
    'providers' => [...],
    'models'    => [...],
    'defaultRun' => [
        'maxTurns'        => 40,
        'maxToolCalls'    => 30,
        'deadlineSeconds' => 600,
        'params'          => ['temperature' => 0.2, 'maxTokens' => 8000],
    ],
]);
```

```php
$options = $registry->runOptions();          // дефолты каталога уже внутри
$options->model = $modelKey;                 // остаётся то, что зависит от запуска
$options->extraParams = ['session_id' => $runId];

$result = (new Runner($orchestra))->run($messages, $toolbox, $systemPromptFn, $options);
```

Каждый вызов `runOptions()` отдаёт новый объект: опции принадлежат прогону, делить их между прогонами нельзя. Неизвестный ключ в `defaultRun` — ошибка сборки каталога, а не молчаливый пропуск. `new RunOptions()` тоже работает — тогда все значения берутся из дефолтов класса.

## Поля

**Какой моделью работать**

- **`model`** (`?string`, по умолчанию `null`) — ключ модели каталога. `null` — модель каталога по умолчанию.
- **`fallback`** (`?string[]`, `null`) — цепочка запасных моделей на этот прогон. `null` — цепочка каталога.
- **`maxSwitches`** (`?int`, `null`) — сколько переключений на запасную модель разрешено на одно обращение к модели, то есть в пределах оборота цикла, а не всего прогона. `null` — значение каталога.
- **`policy`** (`?ErrorPolicy`, `null`) — политика повторов на этот прогон. `null` — политика модели или каталога.
- **`stickyFallback`** (`bool`, `true`) — после переключения продолжать прогон на той модели, которая ответила.

**Границы прогона**

- **`maxTurns`** (`int`, `40`) — сколько раз можно обратиться к модели. Один оборот — один запрос.
- **`maxToolCalls`** (`int`, `30`) — сколько инструментов можно исполнить за весь прогон, а не за оборот.
- **`deadlineSeconds`** (`?float`, `null`) — предельная длительность прогона в секундах. `null` — без срока.

**Что уходит в запрос**

- **`params`** (`GenerationParams`, пустые) — `temperature`, `topP`, `maxTokens`, `seed` на этот прогон.
- **`extraParams`** (`array`, `[]`) — дополнительные поля payload каждого запроса прогона.
- **`toolChoice`** (`string|array`, `'auto'`) — `'auto'`, `'required'`, `'none'` или конкретный инструмент.

**Прочее**

- **`toolArgsGuard`** (`?ToolArgsGuard`, включён) — проверка аргументов на протёкшую разметку вызова. `null` — не проверять.
- **`exposeToolExceptions`** (`bool`, `false`) — показывать ли модели сообщение исключения, вылетевшего из инструмента (обрезанное до 300 символов). По умолчанию модель видит нейтральный текст, подробности идут в лог.
- **`limitNudgeMessage`** (`string`) — сообщение, которое добавляется в историю при исчерпании `maxToolCalls`.
- **`limitFallbackText`** (`string`) — ответ, если добивка вернула ход без текста, но с вызовами инструментов. Пустой ход — это ошибка `empty_response`, а не заглушка.
- **`turnsExhaustedText`** (`string`) — ответ при исчерпании `maxTurns`.
- **`toolLimitReachedText`** (`string`) — что получит модель вместо результата вызова, отклонённого исчерпанным `maxToolCalls`.
- **`toolFailedText`** (`string`) — что получит модель вместо результата инструмента, упавшего с исключением.
- **`toolFailedPrefix`** (`string`) — начало ответа, когда сообщение исключения показывают модели (`exposeToolExceptions`).
- **`encodeFailedText`** (`string`) — что получит модель, если результат инструмента не удалось закодировать в JSON.
- **`firstUseResultKey`** (`string`, `'result'`) — ключ, под который убирается результат-список, когда к нему добавляют пояснение первого вызова.

Все поля публичные — задаются напрямую:

```php
use Hameleon2x\Llm\Agent\Dto\RunOptions;

$options = new RunOptions();
$options->model = 'glm-4.6';
$options->maxTurns = 16;
$options->maxToolCalls = 12;
$options->params->temperature = 0.2;
$options->params->maxTokens = 8000;
```

## `model` и переключение

Ключ ищется в каталоге. При сбое исполнитель повторяет вызов, а затем переходит к следующей модели цепочки; со `stickyFallback = true` оставшиеся обороты идут уже на ответившей модели — возвращаться к упавшей смысла нет.

## `maxTurns` — что такое оборот

Один оборот — один запрос к модели:

1. Собрать системный промт.
2. Один раз вызвать модель.
3. Если вызовов инструментов нет — вернуть успех.
4. Иначе исполнить все вызовы хода и начать следующий оборот.

Несколько вызовов инструментов в одном ходе — это **один** оборот, но несколько единиц из `maxToolCalls`.

## `maxToolCalls` и добивка

Счётчик уменьшается на каждом исполненном вызове по всем оборотам. Когда он обнуляется посреди хода:

1. Оставшиеся вызовы этого хода закрываются ошибкой — история остаётся валидной (у каждого вызова есть ответ).
2. В историю добавляется `Message::user($options->limitNudgeMessage)`.
3. Делается ещё один запрос **без** инструментов.
4. Непустой ответ отдаётся как успех. Ход без текста, но с вызовами инструментов даёт `limitFallbackText`; полностью пустой ход — ошибку `empty_response`.

Итог помечается `Finish::TOOL_LIMIT`, расход токенов добивки входит в `Result::$usage`. Если сам запрос-добивка не удался, прогон возвращает ошибку с категорией и `Finish::ERROR` — заглушка вместо неё не подставляется.

## `deadlineSeconds`

Проверяется перед каждым оборотом. При исчерпании прогон возвращает `Result` с ошибкой категории `deadline`, `finish = Finish::DEADLINE` и полной историей — накопленные результаты инструментов не теряются. Оборот не начинается и тогда, когда до срока осталось меньше секунды: таймаут запроса всё равно округляется вверх до секунды, и такой оборот вернул бы таймаут модели вместо честного «истёк срок».

Срок удерживается и внутри оборота: остаток времени уходит исполнителю как потолок ожидания на этот вызов, поэтому повторы и переключения моделей не уводят прогон за дедлайн. Таймаут очередного запроса при этом тоже не может превысить остаток срока.

## `params` и `extraParams`

`params` перекрывает параметры модели и каталога (слияние по явности), а `unsupported` модели вырезает то, что она не принимает, поверх всего. `extraParams` сливается с полями провайдера и модели и уходит в **каждый** запрос прогона, включая добивку по лимиту:

```php
$options->extraParams = [
    'session_id' => 'agent_42_run_17',   // группировка прогона в observability провайдера
];
```

Стандартные поля (`model`, `messages`, `temperature`, `top_p`, `max_tokens`, `tools`, `tool_choice`, `seed`, `stream`) через `extraParams` не переопределяются.

Плагины OpenRouter — частный случай дополнительных полей:

```php
$options->extraParams['plugins'] = [
    ['id' => 'web', 'max_results' => 5],
];
```

## `toolChoice`

Передаётся в OpenAI-совместимый параметр `tool_choice` как есть.

```php
$options->toolChoice = 'auto';     // решает модель
$options->toolChoice = 'required'; // модель обязана вызвать инструмент
$options->toolChoice = 'none';     // инструменты видны, но вызывать нельзя
$options->toolChoice = ['type' => 'function', 'function' => ['name' => 'get_weather']];
```

## `toolArgsGuard`

Проверяет аргументы перед исполнением инструмента и отклоняет вызов, если в значениях протекла разметка формата вызова. Включён по умолчанию: пропущенная утечка означает исполнение на неполных данных, а ложное срабатывание стоит одного переотправленного вызова.

```php
$options->toolArgsGuard = ToolArgsGuard::default(['~<my_tag~']);  // плюс свои паттерны
$options->toolArgsGuard = null;                                    // выключить
```

## Что возвращает прогон

```php
$result->success;        // bool
$result->content;        // ?string
$result->error;          // ?ErrorInfo — категория сбоя
$result->finish;         // Finish::COMPLETED | TOOL_LIMIT | TURNS_EXHAUSTED | DEADLINE | ERROR | SUSPENDED
$result->messages;       // Message[] — полная история без системного сообщения
$result->turnsUsed;
$result->toolCallsUsed;
$result->usage;          // токены, стоимость, разбивка по моделям
$result->modelKey;       // модель, работавшая последней
$result->attempts;       // AttemptLog[] — попытки, повторы, переключения
$result->lastResponse;   // ?Response — extra, raw, finishReason последнего хода
$result->suspended;      // прогон на паузе, ждёт внешнего ввода
$result->pendingToolCallIds;  // id вызовов, чьи результаты нужно предоставить
```
